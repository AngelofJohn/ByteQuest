## Overview

This document defines the pedagogical approach for ByteQuest's French language curriculum. It covers what we teach, when we teach it, and how we structure the learning experience.

### Goals

- **Effective Learning** - Players actually retain vocabulary and grammar
- **Contextual Teaching** - Language tied to meaningful game situations
- **Progressive Difficulty** - Smooth ramp from beginner to intermediate
- **Engagement Over Anxiety** - Learn through adventure, not pressure

---

## Teaching Philosophy

### Core Principles

| Principle | Description |
|-----------|-------------|
| **Comprehensible Input** | New content just slightly above current level |
| **Meaningful Context** | Words learned through story/quests, not isolated flashcards |
| **Active Recall** | Questions require retrieval, not just recognition |
| **Spaced Repetition** | Review at optimal intervals for long-term retention |
| **Low Stakes** | HP damage for mistakes, but always recoverable |
| **Immediate Feedback** | Know instantly if answer was right/wrong |

### What We Avoid

| Anti-Pattern | Why |
|--------------|-----|
| **Streak Anxiety** | No punishing daily streak mechanics |
| **Gamification Treadmill** | Progress tied to real learning, not arbitrary points |
| **Rote Memorization** | Words in context, not pure translation drills |
| **Information Overload** | One concept at a time, just-in-time teaching |
| **Fear of Failure** | Mistakes are learning opportunities, not punishment |

### Learning Through Play

Rather than interrupting gameplay for lessons, language learning IS the gameplay:

- **Combat = Vocabulary Tests** - Answer correctly to deal damage
- **Dialogue = Reading Comprehension** - NPCs speak in progressively more French
- **Quests = Grammar Application** - Complete objectives using grammar rules
- **Exploration = Discovery** - Find new words in the world naturally

### Zone-Based Progression

Each zone introduces vocabulary and grammar appropriate to its theme:

| Zone | Theme | Vocabulary Focus | Grammar Focus |
|------|-------|------------------|---------------|
| **Dawnmere** | Settlement, basics | Greetings, numbers, colors, family | Articles, être/avoir |
| **Haari Fields** | Farming, nature | Farm animals, crops, weather | Present tense regular verbs |
| **Miner's Deep** | Work, materials | Materials, tools, directions | Prepositions, adjective agreement |

## Grammar Sequencing

### Introduction Order

Grammar concepts are introduced in this sequence:

| Order | Concept | When | Why First |
|-------|---------|------|-----------|
| 1 | Definite Articles (le, la, les) | Level 1 | Foundational |
| 2 | Indefinite Articles (un, une) | Level 2 | Build on articles |
| 3 | Être (present) | Level 2 | Most common verb |
| 4 | Avoir (present) | Level 3 | Second most common |
| 5 | Subject Pronouns | Level 3 | Needed for conjugation |
| 6 | Regular -er Verbs | Level 5 | Largest verb class |
| 7 | Adjective Agreement | Level 7 | Gender/number |
| 8 | Regular -ir/-re Verbs | Level 10 | Expand verb types |
| 9 | Prepositions | Level 12 | Location, direction |
| 10 | Passé Composé (with avoir) | Level 15 | First past tense |
| 11 | Passé Composé (with être) | Level 17 | Movement verbs |
| 12 | Irregular Verbs | Level 18+ | Common exceptions |
| 13 | Imparfait | Level 22 | Past description |
| 14 | Future Tense | Level 25 | Planning, prediction |
| 15 | Conditional | Level 28+ | Hypothetical |

### Grammar Lesson Structure

Each grammar concept follows this teaching pattern:

1. **Introduction** - NPC explains the rule with simple examples
2. **Recognition** - Player identifies correct forms (multiple choice)
3. **Production** - Player constructs sentences using the rule
4. **Integration** - Rule appears naturally in regular lessons
5. **Reinforcement** - Spellbook available for reference

### Grammar Quest Example Flow

```
Quest: "Learning to Be" (Être introduction)

Step 1: Sage Aldric explains être conjugation
Step 2: Match pronouns to conjugations (je → suis, tu → es)
Step 3: Complete sentences: "Je ___ content"
Step 4: Translate: "I am happy" → "Je suis content"
Step 5: Quest complete, être added to Spellbook
```

## Question Type Strategy

### Question Types by Purpose

| Type | Purpose | When to Use | % of Questions |
|------|---------|-------------|----------------|
| **Multiple Choice (FR→EN)** | Recognition | New vocabulary | 30% |
| **Multiple Choice (EN→FR)** | Production | Reinforcement | 25% |
| **Sentence Reorder** | Sentence structure | Grammar practice | 20% |
| **Fill in Blank** | Contextual recall | Mid-lesson | 15% |
| **Free Input** | Full production | Review, mastery | 10% |

### Question Type Progression

```
New Word Introduction:
  Lesson 1: Multiple choice (4 options, FR→EN)
  Lesson 2: Multiple choice (4 options, EN→FR)
  Lesson 3: Fill in blank (with hint)

Reinforcement Phase:
  Review 1: Multiple choice (harder distractors)
  Review 2: Sentence reorder
  Review 3: Fill in blank (no hint)

Mastery Phase:
  Review 4+: Free input (typing)
  Boss encounters: Mixed types, no hints
```

### Distractor Selection

Wrong answers are chosen strategically:

| Level | Distractor Strategy |
|-------|---------------------|
| Beginner | Unrelated words (chat, maison, rouge) |
| Elementary | Same category (chat, chien, vache) |
| Intermediate | Similar spelling (chat, château, chaud) |
| Advanced | False friends, near-synonyms |

## Spaced Repetition Design

### Review Intervals

Words are scheduled for review based on performance:

| Correct Streak | Next Review |
|----------------|-------------|
| 1 | 1 day |
| 2 | 3 days |
| 3 | 7 days |
| 4 | 14 days |
| 5 | 30 days |
| 6+ | 60 days |

### Review Integration

Reviews are woven into regular gameplay:

- **Quest Lessons** - 30% review words, 70% new/quest-specific
- **Review Sessions** - Dedicated review when queue is large
- **Random Encounters** - Quick pop-up reviews during exploration
- **Boss Preparation** - Review relevant vocabulary before exams

### Mastery Tracking

| Status | Criteria | Visual |
|--------|----------|--------|
| **New** | Never answered correctly | ○ |
| **Learning** | 1-2 correct | ◔ |
| **Reviewing** | 3-4 correct | ◑ |
| **Known** | 5+ correct in a row | ◕ |
| **Mastered** | Known for 30+ days | ● |

## Data Optimization Proposal

### Current Issues

The current vocabulary system has scaling problems:

```javascript
// Current: Lots of redundant metadata
{ french: "la mère", english: "the mother", audio: null, hint: "Think 'maternal'" },
{ french: "le père", english: "the father", audio: null, hint: "Think 'paternal'" },
```

- Each entry duplicates article (la/le)
- Audio always `null` (placeholder)
- Hints repeated for every entry
- Hard to query by category, difficulty, or part of speech

### Proposed: Normalized Data Structure

**Core Principles:**
1. **Normalize data** — Each piece of info stored once, referenced by ID
2. **Flat query layer** — Easy to ask "give me all food vocabulary"
3. **Metadata separation** — Separate concerns (translations, hints, difficulty)
4. **Quest mapping** — Track which words appear in which content

**New Structure:**
```
ByteClaude/data/
├── vocabulary/
│   ├── vocab-core.js       # All vocabulary entries (normalized)
│   ├── vocab-metadata.js   # Categories, difficulty, parts of speech
│   └── vocab-index.js      # Query engine
├── grammar/
│   ├── grammar-core.js     # Pronouns, conjugation templates
│   ├── grammar-verbs.js    # Verb database (all forms)
│   └── grammar-index.js    # Query engine
└── content/
    └── quest-mapping.js    # Which words appear in which quests
```

### Normalized Vocabulary Example

```javascript
// vocab-core.js
const VOCABULARY_ENTRIES = {
  // Format: id: [french, article, englishTranslations[], partOfSpeech]
  'V_family_mother': ['mère', 'la', ['mother', 'mom'], 'noun'],
  'V_family_father': ['père', 'le', ['father', 'dad'], 'noun'],
  'V_food_bread': ['pain', 'le', ['bread'], 'noun'],
  'V_verb_to_eat': ['manger', null, ['to eat'], 'verb'],
};

// vocab-metadata.js
const VOCAB_METADATA = {
  difficulty: {
    'V_family_mother': 1,
    'V_verb_to_harvest': 2,
  },
  categories: {
    'family': ['V_family_mother', 'V_family_father', ...],
    'farming': ['V_farm_cow', 'V_farm_field', ...],
  },
  hints: {
    'V_family_mother': "Think 'maternal'",
  },
  quests: {
    'Q001_family': ['V_family_mother', 'V_family_father', ...],
  },
};
```

### Query Engine

```javascript
class VocabularyIndex {
  getByCategory(category) { ... }
  getByDifficulty(difficulty) { ... }
  getForQuest(questId) { ... }
  getForBossTheme(theme) { ... }
  getHint(wordId) { ... }
  getFiltered(filters) { ... }
}
```

### Benefits

1. **Query Flexibility** — Ask for "all pastoral vocabulary" or "all verbs in present tense"
2. **Better Caching** — Precompute question sets
3. **Scaling** — Adding 1000 words doesn't blow up file size
4. **Maintainability** — Changes to one word don't affect others
5. **Boss Integration** — Easy themed vocabulary pools
6. **Analytics** — Track which words appear where

### Migration Path

| Phase | Tasks |
|-------|-------|
| **Phase 1** | Create new system alongside old, no breaking changes |
| **Phase 2** | Migrate lesson content to new IDs |
| **Phase 3** | Deprecate and remove old system |

**Status:** Deferred to Phase 3+
